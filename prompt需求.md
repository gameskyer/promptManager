# AI 绘画提示词管理客户端 (PromptMaster) 需求规格说明书 v2.0

## 1. 项目定位
一款基于 Go 1.24 和 Wails 开发的桌面级客户端，旨在为 AI 绘画爱好者提供结构化的提示词（Prompt）管理、智能组合、版本控制，以及基于 AI 大模型的自动拆解入库功能。

核心设计理念：**零摩擦创作流** —— 版本控制应隐形化，不打断用户的心流状态。

---

## 2. 技术栈规划

| 层级       | 技术选型                          | 说明                        |
| ---------- | --------------------------------- | --------------------------- |
| 后端       | Go 1.24                           | 核心逻辑与本地服务          |
| 前端       | Vue 3 + Tailwind CSS              | 响应式界面                  |
| 本地数据库 | SQLite (GORM)                     | 主数据存储                  |
| 全文检索   | Bleve + 拼音分词插件              | 支持中英文、拼音模糊搜索    |
| AI 驱动    | OpenAI/DeepSeek API 或本地 Ollama | 语义拆解与优化              |
| 版本控制   | 自研差异算法 + 本地快照           | 无 Git 依赖的轻量化版本管理 |

## 3. 界面布局与交互设计 (UI/UX)

### 3.1 整体布局
采用"顶部-左侧-中间-右侧抽屉-底部时间线"五区设计：

**顶部栏 (Top Bar)**：全局搜索框（支持拼音/中英）、API 配置、快速保存（Ctrl+S）

**左侧导航栏 (Side Menu)**：树形多级菜单
- 一级：大类（提示词库、预设库、版本归档）
- 二级：详细分类

**中间内容区 (Main Content - 60% 宽度)**：
- **细化导航区 (Top Tabs)**：基于左侧选中的二级分类，展示子级细化分类（如：人物 -&gt; 发型、眼睛）
- **原子词网格**：紧凑卡片（含中文、英文、前 3 条近义词），标记"已添加"状态

**右侧组合工作区 (Workbench Drawer - 25% 宽度)**：
- 可伸缩列表：原子词条（原词、翻译、上移/下移/权重调节/移除）
- **版本控制条**：显示当前预设版本号（V3）、保存按钮、历史展开开关

**底部时间线抽屉 (Version Timeline - 高度可收缩)**：
- 默认收起，点击右侧"版本历史"展开
- **无文字描述设计**：仅展示缩略图、版本号（V1/V2/V3）、时间戳、星标状态
- 紧凑 Diff 徽章：如 `[+2/-1]` 表示增删词数

## 4. 核心功能需求

### 4.1 原子词与近义词管理
- **正/负区分**：显式区分提示词类型（Positive/Negative）
- **近义词库**：JSON 字符串存储近义词数组，搜索命中近义词即展示该原子词
- **使用统计**：记录各原子词被使用的频次与最近使用时间

### 4.2 组合预设与参数
- **手动参数**：Checkpoint、LoRA 强度、采样参数（Sampler/Steps/CFG Scale）
- **多图预览**：每个预设支持绑定最多 10 张本地预览图
- **版本绑定**：每个版本快照可独立绑定预览图（方便对比不同参数的效果图）

### 4.3 版本控制系统 (Version Control)
**设计原则**：零输入负担，纯视觉识别，自动差异追踪。

#### 4.3.1 快照机制
- **自动保存**：点击保存按钮（Ctrl+S）时，静默创建新版本，**无弹窗、无描述输入**
- **全量快照**：每个版本保存完整的 Prompt 文本、参数、预览图路径
- **版本号**：自动递增（V1, V2, V3...），不支持自定义命名

#### 4.3.2 时间线界面（Timeline）

┌────────────────────────────────────────────┐ 

│ 版本历史                                   │ 

├────────────────────────────────────────────┤

│ 现在（当前编辑）                          │

│  ─────────────────────────────────────    │ 

│  ⭐ V5  14:32  [缩略图]  [+1/-2]          │ 

│  ⚪ V4  11:20  [缩略图]  [+0/-0]          │

│  ⚪ V3  昨天   [缩略图]  [+3/-1]          │ 

│  ⚪ V2  前天   [缩略图]                   │ 

│  ⚪ V1  前天   [缩略图]                   │

└────────────────────────────────────────────┘

- **视觉识别**：依靠缩略图和时间戳定位版本
- **星标系统**：用户可对关键版本（如"最终交付版"）点击星标（⭐），星标版本置顶高亮
- **紧凑徽章**：`[+2/-1]` 表示该版本相比上一版增加了 2 个原子词，删除了 1 个

#### 4.3.3 Diff 对比视图
选中两个版本（如 V1 与 V3）进入对比模式：
- **三栏布局**：左栏 V1 | 中栏变更摘要（自动生成的原子词级 Diff） | 右栏 V3
- **颜色编码**：
  - 🟢 绿色：新增原子词
  - 🔴 红色：删除原子词  
  - 🟡 黄色：参数变更（如 Steps: 20→30）
- **悬浮提示**：鼠标悬停时间线某版本时，浮层显示该版本的紧凑 Diff 摘要（如：`+sitting, -standing`）

#### 4.3.4 版本操作
- **回滚**：双击历史版本，基于该状态创建新版本（如从 V3 回滚到 V2，实际创建 V4 复制 V2 状态，保留 V3 历史）
- **Fork**：右键版本选择"基于此创建新预设"，生成独立分支
- **清理**：支持设置保留策略（如仅保留最近 20 个版本，或仅保留星标版本）

### 4.4 AI 智能拆解 (AI-Powered Exploder)
- **文本拆解**：粘贴长段 Prompt，AI 分析拆解为原子词
- **入库逻辑**：
  - 库中已存在：以库为准，仅补充新中文解释至近义词列表
  - 库中不存在：识别为新词，引导用户确认分类后入库
- **版本感知**：拆解修改后的保存操作自动触发版本快照（V1→V2）

### 4.5 辅助增强功能
- **图像反推 (WD14/Tagger)**：拖拽上传已生成图片，自动反推原子词并建议入库
- **数据迁移**：支持导入 ComfyUI/SD WebUI 的 styles.csv、Civitai 模型页面抓取
- **A/B 测试模式**：选中两个版本，一键生成对比参数表（相同 Seed，不同 Prompt）

## 5. 数据库设计 (Schema)

### 5.1 原子词表 (atoms)

| 字段          | 类型    | 说明                      |
| ------------- | ------- | ------------------------- |
| `id`          | INTEGER | 主键                      |
| `value`       | STRING  | 英文原词 (Unique Index)   |
| `label`       | STRING  | 中文展示标签              |
| `synonyms`    | TEXT    | JSON 字符串存储近义词数组 |
| `type`        | STRING  | Positive / Negative       |
| `category_id` | INTEGER | 关联分类 ID（最细层级）   |
| `usage_count` | INTEGER | 使用频次统计              |

### 5.2 分类表 (categories) - 自关联设计

| 字段         | 类型    | 说明                              |
| ------------ | ------- | --------------------------------- |
| `id`         | INTEGER | 主键                              |
| `name`       | STRING  | 分类名称（如：人物、发型）        |
| `parent_id`  | INTEGER | 父类 ID（0 或 NULL 表示一级分类） |
| `type`       | STRING  | ATOM (原子词) / PRESET (预设)     |
| `sort_order` | INTEGER | 排序权重                          |

### 5.3 预设主表 (presets)

| 字段              | 类型      | 说明                       |
| ----------------- | --------- | -------------------------- |
| `id`              | INTEGER   | 主键                       |
| `title`           | STRING    | 预设标题                   |
| `current_version` | INTEGER   | 当前最新版本号（缓存字段） |
| `created_at`      | TIMESTAMP | 创建时间                   |
| `updated_at`      | TIMESTAMP | 最后修改时间               |
| `is_deleted`      | BOOLEAN   | 软删除标记（回收站机制）   |

### 5.4 预设版本表 (preset_versions) ⭐新增核心表

| 字段             | 类型      | 说明                                                         |
| ---------------- | --------- | ------------------------------------------------------------ |
| `id`             | INTEGER   | 主键                                                         |
| `preset_id`      | INTEGER   | 关联 presets 表（索引）                                      |
| `version_num`    | INTEGER   | 版本号，自增（V1=1, V2=2）                                   |
| `snapshot`       | TEXT      | **JSON** 存储完整快照（pos_text, neg_text, params, preview_paths） |
| `thumbnail_path` | STRING    | 该版本绑定的预览图路径（视觉识别关键）                       |
| `is_starred`     | BOOLEAN   | 用户星标标记（重要版本标识）                                 |
| `created_at`     | TIMESTAMP | 创建时间（唯一识别依据）                                     |
| `diff_stats`     | STRING    | 紧凑变更统计（如 `+2/-1`）                                   |

### 5.5 预览图表 (previews)

| 字段         | 类型      | 说明                              |
| ------------ | --------- | --------------------------------- |
| `id`         | INTEGER   | 主键                              |
| `preset_id`  | INTEGER   | 关联预设                          |
| `version_id` | INTEGER   | 可选关联特定版本（NULL 表示通用） |
| `file_path`  | STRING    | 本地图片路径                      |
| `created_at` | TIMESTAMP | 添加时间                          |

## 6. 业务开发约束

### 6.1 分类读取逻辑
- 左侧一级菜单：`parent_id = 0`
- 左侧二级菜单：`parent_id = [一级ID]`
- 中间细化导航：`parent_id = [左侧二级ID]`
- 搜索体验：Bleve 引擎配置拼音分词（如输入 "fz" 匹配 "服装"）

### 6.2 版本控制约束
- **无描述原则**：禁止要求用户输入版本变更说明，依赖自动 Diff 和缩略图识别
- **快照完整性**：每个版本必须包含完整的 Prompt 文本与参数，禁止差异链式依赖（防止单点损坏）
- **存储优化**：预览图文件不重复存储，多版本引用相同文件路径，除非用户专为某版本上传新图

### 6.3 生成逻辑
- 生成弹窗仅提供文本展示与"一键复制"功能，**不直接调用 AI 生成接口**
- 生成前自动检查：Prompt 长度超限警告、重复词汇提示

### 6.4 数据安全
- API 密钥存储：调用系统密钥链（macOS Keychain / Windows Credential / Linux Secret Service）
- 自动备份：支持设置定时导出加密备份包（含完整版本历史）

### 6.5 性能要求
- 虚拟滚动：原子词网格超过 5000 条时启用虚拟滚动
- 版本加载：时间线展开时懒加载缩略图，仅加载前 20 个版本的完整数据
- Diff 计算：对比视图采用惰性计算，缓存最近的 Diff 结果

## 7. 关键交互流程示例

### 7.1 修改并保存新版本
1. 用户在右侧工作区将 `standing` 修改为 `sitting`
2. 按 Ctrl+S 或点击"保存"
3. 系统静默创建 V2：
   - 生成 JSON 快照
   - 解析原子词变化：`+sitting`, `-standing`
   - 记录 `diff_stats` 为 `+1/-1`
   - 若工作区有预览图，绑定到 V2
4. 底部时间线自动刷新，显示 V2 条目（带缩略图和时间戳）

### 7.2 回滚与对比
1. 用户展开底部时间线，看到 V1（想要恢复的状态）
2. 双击 V1，系统提示"基于 V1 创建新版本 V3？"
3. 确认后创建 V3（V1 的完整复制），当前工作区加载 V3 状态
4. 用户选中 V2 和 V3，点击"对比"
5. 弹出 Diff 视图，中间栏显示：`🟢 +standing, 🔴 -sitting`
